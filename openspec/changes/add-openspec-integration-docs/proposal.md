 # Change: OpenSpec 整合與文件強化

 ## Why
 將專案的 OpenSpec 文檔與使用說明補齊，讓後續以 Spec‑Driven Development 方式穩定演進，並利於新人上手與審閱。

 ## What Changes
 - 更新 `openspec/project.md`：目的、技術棧、規範、測試策略、資料來源與限制
 - 補充 `openspec/AGENTS.md`：專案專屬的執行指令與資料位置、UI 行為
 - 建立本提案與任務清單，並新增流程/文件相關的能力說明（delta）

 ## Impact
 - Specs：新增工程流程/文件能力（process）之說明
 - Code：不涉及程式行為變更

 ## Out of Scope
 - 業務功能與模型行為修改（另立提案）
